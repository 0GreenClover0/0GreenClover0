About Jakt

Jakt is an open-source, memory-safe programming language that compiles by transpiling to C++. It was created within the SerenityOS project — a from-scratch Unix-like operating system — and takes inspiration from Swift, including automatic reference counting for memory safety without a garbage collector.

It borrows elements from other languages like Rust, C# and C++, and combines them in a very elegant final form (in my opinion :).

The language is self-hosted: the Jakt compiler itself is written in Jakt. Features include a rich type system with optionals, generics, enums with associated values, pattern matching, try-catch error propagation, closures, and traits.

"Memory-safe, expressive, and self-hosted — built openly as part of SerenityOS."

A taste of Jakt
// Jakt — inspired by C++, Swift, Rust and C#, transpiles to C++ fn greet(name: String) -> bool { println("Hello, {}!", name) return true } enum Shape { Circle(radius: f64) Rectangle(width: f64, height: f64) fn area(this) -> f64 { return match this { Circle(radius) => 3.14159f64 * radius * radius Rectangle(width, height) => width * height } } } fn main() throws { // 'result' is immutable let result = greet(name: "Reader") // 'shape' is mutable mut shape = Shape::Circle(radius: 5.0) println("Area: {}", shape.area()) }

My Contributions

I contributed 100 commits across the compiler pipeline, fixing bugs and adding features primarily in the typechecker — the largest and most complex part of the compiler. Participating in this project meaningfully sharpened my Git and open-source collaboration skills.

  • Typechecker: control flow analysis improvements — if, while, match, and try-catch blocks resolve control flow correctly in all cases.
  • Typechecker: type inference for match expressions and blocks — infers the most compatible common type across arms.
  • Typechecker: enforcement of subclassing compatibility — subclass instances can be assigned to base-class-typed variables.
  • Typechecker: type-hint propagation into array, set, and dictionary literals for better inference.
  • Typechecker: disallowed invalid constructs — break/continue outside loops, None to non-optional, empty enums, and more.
  • Typechecker: argument label and function return-type inference clean-up.
  • Codegen: fixed dereference operators for generic classes and methods; implicit returns for void-returning lambdas.
  • Parser: fixed infinite loops on malformed input; improved span accuracy; allowed ergonomic newline placement.
  • Error reporting: improved source-span display, including first-line and empty-line edge cases.
  • VS Code extension: added syntax highlighting for references, mut, is, override, virtual, import, floats, and more.
